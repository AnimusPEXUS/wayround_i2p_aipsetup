Welcome to aipsetup’s documentation!¶

aipsetup consists of several modules.

aipsetup is full circle desidgen for maintaining GNU/Linux distribution.

aipsetup can create packages.

aipsetup can install and remove packages.

aipsetup can distribute packages and self.

Modules¶

All module names are relative to org.wayround.aipsetup package

  • Package building modules:
  • buildscript
  • buildingsite
  • build
  • pack
  • Building functions from othe modues:
  • package.build()
  • package.complete()
  • Package Installing and Uninstalling modules:
  • package
  • pkgsnapshot - installation environment snapshots
  • Other package tools:
    • pkgdeps
    • pkgindex - index sources, maintain repository
  • Modules for serving and getting packages:
  • server
  • serverui - server UI module
  • client
  • unicorn - manipulate UNICORN installation base dir
  • GNU/Linux system tools:
  • docbook - automated docbook schemas installed
  • clean - system and aipsetup internals cleaning utilities
  • sysupdates - Update system bindings and such
  • Configuration
  • config - aipsetup config tools
  • constitution - host, build, target
  • Package info related modules
  • info - module to work with package info files on disk
  • name - package name parsing functions
  • pkginfo - package info related functions
  • pkglatest - control on information about latest source tarballs or latest asp packages
  • pkgtag - packages tagging functionality
  • version - version comparison utilities
  • Editors
  • infoeditor - edit package info on disk and update pkginfo database
  • latesteditor - latests pacakges editor
  • tageditor - package tags editor
  • aipsetup internals:
  • dbconnections - aipsetup global database connector
  • gtk - aipsetup global gtk connection
  • help - aipsetup help system
  • repoman - gathering aipsetup CLI interface for following modules:
    • pkgindex;
    • pkginfo;
    • pkglatest;
    • pkgtag
